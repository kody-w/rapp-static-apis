# RAPP static industry-template APIs

No-PII, unauthenticated static product adapters for the 64 canonical AIBAST
industry-agent templates. This sub-API conforms to `rapp-static-api/1.0`.

## Source/generated boundary

Hand-maintained or pinned inputs:

- `manifest.json` — build contract, fixed timestamp, bases, and source hash.
- `seed/static_api_catalog.json` — pinned canonical catalog snapshot.
- `build.py`, `validate.py`, `test_industry_templates.py`, this README.

Generated by the one build command:

- `registry.json`
- `api/` — status, schemas, capability routes, collections, records, receipts.
- `connectors/` — one Swagger 2.0 custom-connector definition per adapter.

```bash
python3 industry-templates/build.py
```

The build is a pure, deterministic function of the two inputs. It uses a fixed
timestamp and stable SHA-256-derived IDs.

## Endpoint layout

```text
api/v1/<adapter>/<resource>.json
api/v1/<adapter>/<resource>/records/<record_id>.json
api/v1/<adapter>/receipts/<capability>.json
```

Every endpoint is a static `GET`. Receipt endpoints simulate write outcomes and
never mutate a product. Records use primitive operational fields only; no names,
email addresses, phone numbers, postal addresses, free text, or real identifiers
are present.

## Custom connectors

`connectors/<adapter>.swagger.json` provides one unauthenticated Swagger 2.0
definition per normalized product adapter against `raw.githubusercontent.com`.
Grouping is deliberately one connector per adapter so SAP ECC, SAP S/4HANA,
SAP PM, Epic EHR, Epic Clarity, and other semantically distinct APIs remain
separate.

Every operation has an exact inline response schema. Collection responses type
the envelope and every `value.items` field; record responses type the envelope
and `record` fields; receipt responses use the canonical simulated-write shape.
Required fields and primitive `date`/`date-time` formats come directly from the
pinned 43-schema catalog.

Static-host limitation: record operations resolve only the stable IDs listed in
`registry.json`; an arbitrary unknown ID returns GitHub's normal 404. Receipt
operations are deterministic GET previews, not POST operations.
