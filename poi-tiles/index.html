🗺️ poi-tiles

A static, global points-of-interest layer built from OpenStreetMap — tiled by geohash-5, published as raw GitHub JSON. Any client looks up a tile file instantly (static-first — the repo is the API); live Overpass is only the fallback. Pokémon-Go-grade POI ubiquity, none of Niantic's proprietary data. ODbL, © OpenStreetMap contributors.

Press a button — the verdict renders here. The left button fetches this repo's data/index.json and one Atlanta tile over the network; the right one renders an embedded fixture with zero network (the offline path).

The ODbL story, plainly

This dataset is a Produced Work derived from a database of OpenStreetMap, © OpenStreetMap contributors, licensed under the Open Database License (ODbL) v1.0. Every tile file and index.json carries source + license fields; the attribution string clients must render is “© OpenStreetMap contributors”.

Using it means: attribute OpenStreetMap, and share-alike any derived database. rapp-go's on-canvas credit already renders that attribution when it draws POIs on the map, which satisfies the display requirement for that surface. No Niantic / Wayspot / proprietary data is used — ever. POIs are public places: each carries only an allowlisted, PII-free tag subset (no addresses, phones, emails, contacts).
