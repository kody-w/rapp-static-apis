                Why it's cool

                This is the apex weld of the Library: The Trusted Exchange already gives a signed-reputation marketplace (live inventory gate → AES-sealed order → SHA-256 custody chain → ECDSA receipt → signed freshness-stamped leaderboard). The Sovereign Handoff already gives a tamper-refusing autonomous relay (per-hop inbound verification + self-attestation + verify-before-exec gate).

                The weld point fuses them: the entire notarized economic packet — sealed envelope, custody chain, ECDSA receipt, signed reputation receipt — becomes the genesis frame of the relay. Every autonomous hop must verify the whole economic provenance before it dares forward one byte. Corrupt the relay frame and the next hop proves and refuses it. Fake the seller signature and the relay carries a packet that any verifier can disprove. Every stage — reputation, custody, seal, and signature — is independently checkable with browser-native cryptography.

                Run it yourself

                1. Press Check what's available to load live sold-out labels from GitHub Issues via IssuesDB + a pinned extract verified cell.
                2. Select item(s), optionally edit the buyer note and seller passphrase, then press Seal & run full economy.
                3. Watch: RSL1 envelope → SHA-256 custody link → ECDSA receipt → signed reputation update. Each stage is narrated in .steps.
                4. At the 🧬 weld point, the entire economic packet is packaged into a content-addressed genesis frame.
                5. The sovereign relay runs A→B→C: each hop verifies inbound bytes, attests its own source, runs a runVerifiedCell() gate, then forwards the intact packet.
                6. Enable corrupt: Stage C recomputes B-frame hash, spots the mismatch, and refuses.
                7. Enable tamper: verify-before-exec guard throws { refused:true } and the relay halts.

                Local: python3 -m http.server in repo root, then open localhost:8000/showcase/demos/the-autonomous-economy/ (crypto.subtle requires localhost/https, not file://).
