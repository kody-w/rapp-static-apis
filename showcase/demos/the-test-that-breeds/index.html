      Why it's cool

      A printed square has no CPU, no runtime, no test suite. Yet the instant a phone points at it, the phone becomes the lab: it boots an entire language — CPython 3.12, compiled to WebAssembly — inside a sandbox that can't touch your DOM, cookies, or storage, and it does real science on itself. It doesn't just run a function; it interrogates one — throwing hundreds of random inputs at a property until the function betrays itself, then patiently deleting and shrinking that input until nothing remains but the smallest possible proof of the bug.

      Then the part that makes it breed: it takes that irreducible failure and grows a new artifact from it. The child QR isn't a picture of the answer — it's a complete, self-contained, SHA-256-pinned regression test, and the failing input itself is what determines the child. Scan it and a different stranger's browser re-verifies those exact bytes and re-runs the exact failure — forever, with no backend, no CI, no server keeping it alive. A test discovered a bug and then handed you the next test, printed in ink. Fix the code and the same square flips from FAIL to PASS. The regression suite is the physical world.

      The pattern · fuzz → shrink → freeze → breed

      1. boot — hand the sandboxed /vbrainstem-cell iframe the vbs-init handshake; it loads Pyodide and cold-boots CPython 3.12, headless, no server.
      2. fuzz — op:'eval' a pure-Python property-based fuzzer over a private MessagePort; it random-searches inputs (seeded, deterministic) until decode(encode(xs)) == xs is violated.
      3. shrink — greedily delete elements and walk magnitudes toward zero while the property still fails, until an irreducible counterexample remains.
      4. freeze — emit a self-contained pytest-style test string carrying that exact input, and SHA-256 it (crypto.subtle, via the shared lib's sha256hex) to a content-address.
      5. breed — encode {src, pin} into a fresh QR with qr.mjs and draw it on a <canvas>. The scanner of that child does step-4-in-reverse: recompute the hash, refuse on mismatch, boot, re-run.

      Run it yourself

      1. Open the raw runtime at /vbrainstem-cell — the same sandbox this page embeds — and paste any Python into its eval box.
      2. Swap the target: any encode/decode pair (or any property P(x)) whose bug you want hunted. The fuzzer + shrinker are ~30 lines of pure Python — no hypothesis install, it boots in the browser.
      3. Press Breed the test above, then scan the child QR with your phone — it re-runs the frozen failure on a second device, verify-before-exec, no server anywhere.

      Local: python3 -m http.server in the repo root, then open localhost:8000/showcase/demos/the-test-that-breeds/ (Pyodide + crypto.subtle need a secure context and a network — localhost or https, not file://).

      Reusable pattern:

      The same sandbox runs RAR agents with verify-before-exec (op:'run'): a scannable trigger can ship a whole tamper-checked compute cell — here, one whose output is another verifiable compute cell. That closure (a cell that emits its own successor, content-addressed, no backend) is the primitive: property discovery → frozen artifact → re-runnable trigger, composable forever.
