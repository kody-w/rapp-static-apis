The Artifact With a Signed Bloodline

① Live — grow the signed bloodline (run it right now)

No server, no ledger. Your browser mints a one-time ECDSA P-256 keypair with native crypto.subtle. Each press runs one self-improvement pass — the loop.prompt.md measure → keep-if-better discipline, as code — and records staten = sign( SHA-256(state) ‖ sign-1 ). The signature of each generation covers the signature of the one before it, so the chain is append-only and self-witnessing. It persists in IndexedDB, so your bloodline survives a reload.

      Why it's cool

      Version history is usually a story a server tells you. Git gives you tamper-evident history via parent hashes — but nothing stops whoever hosts it from rewriting the past and handing you a clean-looking tree. This is that idea with a signature welded to every link, and no host at all. Each generation signs SHA-256(state) ‖ sign-1, so the head signature is transitively a commitment to the entire ancestry: change any state, anywhere in the past, and the signature over that generation stops verifying. The page doesn't just say "something's wrong" — it points at the precise generation where the bloodline was cut, because that's the first link whose signature no longer covers its bytes.

      And it's self-building in the honest sense: the thing keeping the record is the same discipline that produces the improvements. Run a pass, and the win is only kept if it's signed into the lineage — an append-only ratchet (loop.prompt.md) where every prior version of the artifact carries cryptographic proof it existed and in what order. The proof is portable: the QR above is the whole trust root — a public key and a signed head — small enough to print, mail, or tattoo. A doubter doesn't take your word for the lineage; they scan it and re-verify it themselves, with their own crypto.subtle, offline.

      Run it yourself

      1. In ①, press Run an improvement pass a few times — watch the score climb and each generation sign the one before it. Reload the page: the bloodline is still there (it lives in IndexedDB, keyed by the same P-256 key).
      2. In ②, hit Verify — the whole chain goes green. Then pick an ancestor generation and press Snap it & re-verify: that link turns red (signature ✗) and everything after it is marked orphaned. Press Restore to prove it heals back to all-green.
      3. Scan the QR with any phone. The payload is { v:"bloodline/1", gens, pub, head, sig } — enough for any WebCrypto (browser, agent, CLI) to import the raw key and confirm this exact head was signed by it.

      Local: python3 -m http.server in the repo root, then open localhost:8000/showcase/demos/the-artifact-with-a-signed-bloodline/ (crypto.subtle + IndexedDB need a secure context — localhost or https, not file://).

      For agents 🤖

      Reusable pattern:

      Compose it: make your state transitions self-witnessing. Each new state signs SHA-256(state) ‖ prev_signature under one ECDSA P-256 key; append the link to a local store (IndexedDB, a file, a content-store). Any party with the public key walks the chain and, on a mutated ancestor, gets the exact index that stops verifying — tamper-evident provenance with no server, ledger, or blockchain. It's the signing twin of /fn verify-before-exec: that proves these bytes are the pinned bytes; this proves these bytes descend, in this order, from a state signed by this key.
