The Page That Proves Itself

Live — self-integrity proof

Your browser just fetched this page's own index.html source, recomputed its SHA-256 with crypto.subtle, and sealed it below. Change one byte anywhere in this file and the hash changes instantly — no server, no library, no trust required.

    Why it's cool

    Every page on the web can claim to be what it says it is — but this one proves it, live, in your browser. The moment the document loads, it fetches its own bytes back from the server, runs them through native browser SHA-256 (crypto.subtle.digest), and displays a hash you can independently verify. No server is asked to vouch for it. No JavaScript library is trusted. The proof is computed entirely inside the same sandbox that rendered the page itself.

    The beautiful constraint: if anyone modifies even a single character of this page — in transit, at the CDN, at the host — the displayed hash changes immediately. Publish the expected hash out-of-band (in a QR code, a signed git tag, another page) and you have a tamper-evident delivery system with zero infrastructure.

    Run it yourself

    1. Clone the repo: git clone https://github.com/kody-w/rapp-static-apis
    2. Serve it locally: python3 -m http.server in the repo root.
    3. Open localhost:8000/showcase/demos/the-page-that-proves-itself/ — the proof runs on load.
    4. Edit any character in index.html, reload — watch the hash change.
    5. Restore the original — the hash returns to the published value.

    crypto.subtle requires a secure context: use localhost or https://, not file://.

    For agents 🤖

    Reusable pattern:

    Pattern: fetch(location.href) → arrayBuffer() → crypto.subtle.digest('SHA-256', buf) → display hex. Anchor the expected hash in a signed manifest, a QR, or a companion verified cell (runVerifiedCell()) to make delivery tamper-evident end-to-end.
