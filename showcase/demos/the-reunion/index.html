    Why it's cool

    Every archive you've ever used answers a conflict by picking a winner: last-write-wins, a merge that clobbers, a "resolve conflict" dialog that throws one side away. A family's memory can't work like that — Aunt May really did remember sun and a 12-pound bass, and Uncle Joe really did remember rain and a card game. Both are true to the person who holds them.

    The Dream Catcher merge (rapp-twin.profile.md §4) obeys one law: additive append + dedupe; NOTHING IS LOST. It content-addresses each memory by SHA-256 — the exact same trust-the-hash primitive that makes verified code tamper-proof in /track's runVerifiedCell, here pointed at data instead of bytecode. Two byte-identical stories fold into one frame (they were never in conflict). A slot collision — same event, different memory — is not a conflict to resolve; it's a contradiction to keep, surfaced as an alternate frame in a parallel dimension. The loser never expires. Because the address is the content, every version is immutable and recoverable by its hash, forever, with no server and nothing to shut down.

    Run it yourself

    1. Open this page over localhost or https (crypto.subtle needs a secure context — not file://).
    2. Swap the two bones objects in the script for your own family's memories. Give two people the same event with different text — watch the archive keep both instead of choosing.
    3. To make it a real, tamper-proof, shareable primitive: the merge is a pure function of the exact /fn shape (deterministic compute over caller-supplied rows). Pin it as a /fn cell and drive it through runVerifiedCell so the merge logic itself is verify-before-act.

    Local: python3 -m http.server in the repo root, then open localhost:8000/showcase/demos/the-reunion/.
