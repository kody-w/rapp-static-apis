WELD: The Living Spec

Live — prove + breed + re-run

This welded flow keeps both donor organs intact: first this page fetches and hashes its own index.html bytes, then a sandboxed CPython fuzzer breeds a minimized conformance test from the same spec rule, pins it by SHA-256, draws a child QR, and re-runs that exact child verify-before-exec.

⚠ first run cold-boots Pyodide from CDN (~10 MB). Use localhost/https and a network.

    Transplant A — The Page That Proves Itself

    The page reuses the donor self-attestation loop exactly: fetch('index.html') → arrayBuffer() → crypto.subtle.digest('SHA-256'). That proof is computed live in your browser and becomes an input to the next stage.

    Transplant B — The Test That Breeds

    The page keeps the donor breeding loop: boot sandboxed CPython, fuzz a property until failure, shrink the counterexample, freeze a self-contained test string, hash-pin it, and draw it as a fresh child QR.

    Weld point — The spec mutates itself safely

    The generated child test embeds the attested page hash and the declared rule. So each run emits a conformance test that checks the claim and carries provenance of the exact spec bytes that bred it.
