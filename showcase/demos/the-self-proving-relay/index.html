The Self-Proving Relay

Live — run it right now

This weld combines three donor moves at every hop: the stage verifies the inbound content-addressed frame, fetches and hashes this page's own source to attest local integrity, then passes through a verify-before-exec code gate. Any mismatch flips to REFUSED and the relay halts.

    Why it's cool

    The relay donor proves upstream payload integrity. The self-proving-page donor proves local runtime integrity. The byte-trap donor proves enforcement by refusing execution on mismatch. Welded together, each hop proves both sides of trust at once: "what I received is untampered" and "what I am is untampered."

    That means a relay can be audited link-by-link with plain hashes, and every stage can halt itself without asking a server. A compromised transit path, cache, or host edit cannot silently propagate — the next hop names the broken hash and refuses cold.

    Run it yourself

    1. Click Run self-proving relay: all three stages complete, each showing frame verification + self-hash attest + guarded run.
    2. Enable corrupt: flip one byte in B→C frame: Stage C recomputes B's frame hash and refuses.
    3. Enable tamper: force code-gate hash mismatch: the stage's verify-before-exec gate throws { refused:true }, and the relay stops immediately.

    Local: python3 -m http.server in the repo root, then open localhost:8000/showcase/demos/the-self-proving-relay/ (crypto.subtle needs a secure context — localhost or https, not file://).
