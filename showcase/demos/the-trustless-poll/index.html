WELD: The Trustless Poll

Live — let signed ballots race into consensus

This keeps both donors alive. First each agent runs the-unbreakable-promise: generate a fresh ECDSA P-256 keypair, sign one canonical ballot, and verify it immediately. Then the signed receipts become the proposals for the-fast-forward-parliament: all remaining ballots fetch the same parent and race one append-only fast-forward update. Exactly one lands per parent; the losers adopt and try again on the next head. The ledger itself becomes the tally.

    Why it's cool

    This is not "trust us, we counted correctly." It is a poll whose counting surface is the append-only ledger itself. Every accepted row carries a ballot message, the public key that signed it, and the signature proving it. The fast-forward rule makes the order tamper-evident: one parent, one winning append. Re-verify the chain and you can prove three things with no coordinator: who cast each ballot, what exact choice they signed, and that no key appears twice.

    So the tally cannot be forged by a host, retracted by editing history, or padded with duplicate votes from the same key. If any byte changes, either signature verification fails or the chain hash snaps.

    Run it yourself

    1. Press Run the trustless poll. The page first verifies a pinned cell from /track, then mints six browser-local ECDSA ballot receipts for the exact poll text.
    2. Watch the signed ballots race the shared append-only head. Exactly one ballot lands per parent, and the rest fast-forward onto the winner before the next round begins.
    3. At the end, the page re-verifies every signature and every parent hash in the accepted order, then derives the tally from that ledger alone.

    Local: python3 -m http.server in the repo root, then open localhost:8000/showcase/demos/the-trustless-poll/ (crypto.subtle + IndexedDB need localhost or https, not file://).
