      ③ Custody stamp — tamper-evident trail

      The sealed order envelope is SHA-256 content-addressed. If any byte of the ciphertext changes in transit the hash won't match and tampering is immediately provable from the genesis anchor — no server required.

              Why it's cool

              This keeps the-notarized-marketplace whole (live stock truth → sealed orders → custody chain → seller-signed receipt), then welds in the-verifiable-tournament whole (signed entry + SHA-256 freshness proof + verified-cell board render). The weld point turns each completed purchase into a signed reputation event on a board that is both attributable and provably current.

              So trust is composable: you don't just trust the listing, or the receipt, or the leaderboard — you can verify every stage independently with browser-native cryptography and pinned compute bytes.

              Run it yourself

              1. Press Check what's available to load live sold-out labels from GitHub Issues.
              2. Select item(s), then press Seal & submit order to produce the RSL1 envelope + custody stamp.
              3. Inspect the emitted seller-signed receipt JSON.
              4. Watch section ⑤ update seller reputation with a new signed payload embedded into a freshness-stamped leaderboard.
              5. Repeat purchases to increase completed orders and see the reputation score rise with new signatures.

              Local: python3 -m http.server in repo root, then open localhost:8000/showcase/demos/the-trusted-exchange/ (crypto.subtle requires localhost/https, not file://).
