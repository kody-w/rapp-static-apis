    Why it's cool

    Every API you've ever used dies when its host dies. Change the domain, delete the repo, let the cert lapse — the endpoint is gone and every caller breaks, because the caller trusted a location. Here the caller trusts a hash. track@3fe23f859268 names an exact 1,784 bytes of code; the /resolver maps that name onto a host in a single line, so kody-w.github.io, someone-else.github.io, a GitLab mirror, a raw CDN, an IPFS gateway, or the laptop in your bag are all just different doors to the same content. Kill any door. Your browser re-derives the SHA-256 from whatever door survives and refuses to run a single altered byte — so you can safely trust an anonymous mirror precisely because you never trusted the host to begin with. The repo can be deleted. The API cannot.

    Run it yourself

    1. Import /resolver's one function: expand('track@3fe23f859268') and expand('anyone-else/track@3fe23f859268') return the same sha8 from different hosts — the whole trick is that the host is one line.
    2. Feed that pin to runVerifiedCell() against any mirror base (this page uses the local /track fork): it fetches the bytes, recomputes SHA-256 in your browser, and throws before running on any mismatch.
    3. Fork this repo to your own account. The same demo.json + index.html now answer from you.github.io — same hash, new host, zero code changes.

    Local: python3 -m http.server in the repo root, then open localhost:8000/showcase/demos/kill-the-host/ — the surviving mirror answering is literally your machine, not GitHub (crypto.subtle needs a secure context — localhost or https, not file://).
