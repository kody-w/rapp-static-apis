The Relay

Live — run it right now

Three simulated agents run inline in your browser. Each one serializes its output to bytes, hashes those bytes with crypto.subtle SHA-256, and passes the frame forward. The receiver recomputes the hash from the raw bytes — if they don't match the pinned sha8, it refuses to act. The same verify-before-act gate that runVerifiedCell uses for code cells, applied here to data frames between agents.

    Why it's cool

    In most agent pipelines the downstream stage trusts the upstream stage — it just reads the payload and acts. That trust is invisible and uncheckable. The Relay makes it explicit and cryptographically enforced: every frame is content-addressed by its SHA-256. Before Stage B touches Stage A's output, it recomputes that hash from the raw bytes. If a single bit changed — a MITM rewrite, a storage glitch, a compromised cache — the hashes diverge and B refuses to act, cold. No quorum, no leader, no certificate authority. Just math.

    This is the same primitive runVerifiedCell uses to gate code execution, applied to data handoffs between agents. A relay of N stages is an integrity chain: you can audit any link by recomputing one hash. The chain is only as strong as its weakest frame — and every frame is verifiable.

    Run it yourself

    1. Click 🔗 Run the relay: watch Agents A → B → C each verify the prior hash (green ✓) and emit their own frame.
    2. Check corrupt: flip a byte of Stage B's frame and run again: Agent C recomputes B's hash, finds a mismatch, and refuses — the bad-stage border turns red and the relay stops there.
    3. To use this pattern in your own agent: serialize your output with JSON.stringify, hash the bytes with sha256hex (exported from ../../lib/showcase.js), and send { payload, bytes, sha8 }. The receiver re-hashes and refuses on any mismatch before acting.

    Local: python3 -m http.server in the repo root, then open localhost:8000/showcase/demos/the-relay/ (crypto.subtle needs a secure context — localhost or https, not file://).
