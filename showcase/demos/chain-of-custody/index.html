    Why it's cool

    Most integrity demos prove only the current file. This proves the whole timeline. Each version stores its own content hash and the parent hash it claims to descend from. Starting from one trusted genesis hash, a stranger can replay the full lineage in their browser. If any old byte is edited, descendants no longer connect and the verifier points to the first broken hop.

    That distinction matters: freshness says “this is current”; chain of custody says “this came through these exact historical states.”

    Run it yourself

    1. Click Verify full custody chain on the sample record. You should get a green proof and a head hash.
    2. Click Flip one byte in v2, then verify again. The chain should fail exactly at version 2.
    3. Paste your own chain JSON with genesis and versions[] links; the page replays everything client-side.

    Local: python3 -m http.server in the repo root, then open localhost:8000/showcase/demos/chain-of-custody/ (crypto.subtle needs a secure context — localhost or https, not file://).

    For agents 🤖

    Reusable pattern:

    Use a single trusted genesis hash, then verify each link as: sha256(content) == hash and parent == previous hash. Stop at first mismatch and report the exact broken generation.
