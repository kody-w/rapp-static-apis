    Why it's cool

    The-unriggable-ticket proves nobody can predetermine a ticket outcome: the deciding number lives inside a global randomness beacon that thousands of nodes co-produce every 3 s — it literally does not exist when the ticket is issued. Proof-of-freshness proves a displayed value came from bytes fetched right now, not staged ahead of time.

    The weld point is that the draw itself gains both guarantees simultaneously. The freshness hash anchors the draw to the moment it was run — so the operator cannot stage a "draw" that was really computed yesterday. The drand randomness anchors the draw to a public beacon that was unknowable at ticket issuance — so no ticket can be pre-screened for a win. A raffle that is simultaneously unriggable (future public randomness) and unpredictable before the draw (live freshness proof) is something neither donor could achieve alone.

    Draw formula: draw_entropy = SHA-256(freshness_hash ":" drand_randomness) · winner = tickets[ BigInt("0x" + draw_entropy) mod n ]

    Run the draw above to generate the verification script.

    The drand randomness for the committed round is permanently fixed and public. The freshness hash changes every run (it reflects the exact live bytes your browser fetched), which is the proof the draw is genuinely fresh — not backdated. Anyone who runs the draw at a different moment gets a different freshness hash and therefore a different winner, but the inputs are fully auditable: the hash is your receipt.

    Local: python3 -m http.server in the repo root, then open localhost:8000/showcase/demos/the-provable-raffle/ (crypto.subtle needs a secure context — localhost or https, not file://).
