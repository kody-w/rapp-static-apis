The Brainstem Baton

Live — run it right now

Two separate CPython runtimes boot in two sandboxed, opaque-origin iframes on this page — call them Machine A and Machine B. Press the button: Machine A runs a resumable agent for a few rounds and freezes it mid-thought (pickle.dumps of its live state). Your browser seals those bytes with AES-256-GCM and gives the sealed frame a content address (its SHA-256). Machine B — a completely different runtime — recomputes that hash (refusing on any mismatch), unseals with the shared key, pickle.loads, and continues from the exact round A stopped on. No server ever sees the mind in flight.

⚠ first run cold-boots Pyodide/CPython twice (~10 MB from a CDN, a few seconds; the second boot is served from browser cache). Needs a network + a secure context (localhost or https, not file://). It caches after the first run.

Watch what's happening

    Why it's cool

    Every "resume where you left off" you've ever used goes back to a server: the state lives in someone's database, and you reconnect to their machine. This doesn't. Machine A's running agent — its counter, its accumulator variables a and b, its whole trail of thought — is frozen into a handful of bytes, sealed, given a name that is its own hash, and dropped as a single static file. Machine B is a different CPython process that has never talked to A. It picks up that file, proves it wasn't altered in flight (the recomputed SHA-256 must equal the address, or it refuses before touching it), opens the seal with the shared key, and the agent keeps thinking — round 5 follows A's round 4, computed from A's exact variables. That's live process migration between two machines with nothing but a static file between them.

    Flip the tamper box and watch the handoff die: change one byte of the sealed frame and Machine B's recomputed hash no longer matches the address, so it never thaws the mind. The hash guards the handoff; the seal gates who may read it; both refuse in the receiver's own browser, with no host to ask. Honest scope: "the process" here is a serializable continuation — an explicit state dict plus a step() function that both machines already hold — not a raw C stack snapshot. But that is precisely the shape a resumable agent already has: state is data, the next move is code. Migrate the data, and the mind moves.

    The pattern · pickle → seal → address → verify → resume

    1. freeze — Machine A's /vbrainstem-cell (Pyodide CPython in a sandboxed iframe) runs the agent a few rounds, then pickle.dumps(state) → base64.
    2. seal — your browser derives a 256-bit key (PBKDF2·SHA-256) and seals the pickle bytes with AES-256-GCM — the rapp-sealed/1.0 (RSL1) envelope, native crypto.subtle.
    3. address — content-address the sealed frame by its SHA-256 (a /fn-shaped sha8): the file's name is its own fingerprint.
    4. verify — Machine B recomputes that SHA-256 with the shared lib's sha256hex (the exact integrity primitive /track's runVerifiedCell uses) and refuses on any mismatch — verify-before-exec, over live state instead of code bytes.
    5. resume — B unseals, pickle.loads, and calls the same step() it already holds, continuing from A's exact round.

    Run it yourself

    1. Open this page over localhost or https (Pyodide + crypto.subtle need a secure context and a network — not file://), then press Pass the baton.
    2. Read Machine A's frozen trail, then watch Machine B's first term continue it exactly — same counter, same a/b. Then tick tamper and run again: B refuses before it thaws anything.
    3. Make it real: replace the toy agent's state + step() with your own resumable agent, publish the sealed frame to any static store (a gist, a repo, a content-store), and let any second runtime pull it, verify, unseal, and continue. The frame is inert data; the two runtimes never touch.

    Local: python3 -m http.server in the repo root, then open localhost:8000/showcase/demos/the-brainstem-baton/.

    For agents 🤖

    Reusable pattern:

    The mechanism, generalized: serialize the agent's live state (data, not the C stack) → seal it (rapp-sealed/1.0, key gates reads) → content-address it (sha8, the frame's own name) → the receiver recomputes the hash and refuses on mismatch (verify-before-exec) → unseal → loads → step(). Integrity (the hash) and confidentiality (the seal) are two independent planes, each enforced in the receiver's own runtime — no coordinating server. The same /vbrainstem-cell also runs pinned RAR agents with verify-before-exec (op:'run'), so a migrated state and the code that advances it can each be hash-checked before anything executes.
