    Why it's cool

    The Notarized Marketplace donor already gives a sealed, custody-stamped, seller-signed order artifact. The Self-Proving Relay donor already gives per-hop inbound verification, self-attestation, and refuse-on-mismatch enforcement. Weld them and the thing crossing untrusted agents is now a high-value notarized object with provenance checks at every hop.

    If any transit byte flips, the next hop proves and refuses it. If any code-gate bytes mismatch, execution halts. If payload authorship is disputed, the embedded ECDSA receipt still verifies independently. Confidentiality, custody, and authenticity survive the full relay.

    Run it yourself

    1. Click Run sovereign handoff to mint a sealed + custody-stamped + signed order, then relay it A→B→C with per-hop verification.
    2. Enable corrupt: Stage C recomputes B-frame hash, spots mismatch, and refuses before forwarding.
    3. Enable tamper: verify-before-exec guard throws { refused:true } and the handoff halts.

    Local: python3 -m http.server in repo root, then open localhost:8000/showcase/demos/the-sovereign-handoff/ (crypto.subtle needs HTTPS or localhost, not file://).
