    Why it's cool

    A typical leaderboard is a screenshot waiting to be faked. This one has three cryptographic locks that none of the donor demos had alone:

    • Signed entries (from the-unbreakable-promise): each player's slot carries an ECDSA P-256 signature from a keypair that never left their browser. Change one character and verification fails.
    • Freshness proof (from proof-of-freshness): the board is stamped with the SHA-256 of the live ranking payload fetched right now. A stale snapshot has a different hash.
    • Verified compute (from living-leaderboard): runVerifiedCell() re-hashes the pinned extract cell before importing it, so the code that processes the board is as tamper-evident as the data.

    All three proofs compose at the weld point: a signed entry inside a freshness-stamped board processed by hash-verified code. No server runs in the middle. No screenshot can impersonate it.

    Run it yourself

    1. Enter any name and score, press 🧬 Enter Tournament.
    2. Copy the signed receipt JSON at the bottom. Share it anywhere — any stranger can verify your ECDSA signature with crypto.subtle.verify using the embedded public key.
    3. Run it again: the board freshness hash changes whenever the upstream ranking feed changes, proving the board is live and not a cached screenshot.

    Local: python3 -m http.server in the repo root, then open localhost:8000/showcase/demos/the-verifiable-tournament/ (WebCrypto requires a secure context — localhost or https, not file://).
