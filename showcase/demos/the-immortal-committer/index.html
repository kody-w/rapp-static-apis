    Why it's cool

    An engraved square of granite has no CPU, no battery, no network — and neither does the person under it. Yet the instant a mourner points a phone at the stone, their device becomes the machine: it pulls a tiny content-addressed program, proves it wasn't tampered with (the SHA-256 must match the pin, or it refuses to run), and reads the live public count of everyone still starring and forking the deceased's work. The number on the grave is not carved. It is computed, live, in the mourner's own hand — and it is higher today than it was at the funeral. There is no server to pay for, nothing to renew, nothing anyone can shut down. The stone will keep counting as long as one person a decade cares enough to look, and the code that runs it is the same verified cell whether it's scanned tomorrow or in eighty years.

    Make your own · run it yourself

    1. Open /track, point src at any public GitHub repo API URL (https://api.github.com/repos/<owner>/<repo>) and set path to stargazers_count — or subscribers_count, a follower count, a fundraiser total, any CORS-open public number.
    2. Hit Make QR — it encodes track/run.html#<spec> with the zero-dependency encoder.
    3. Etch it in stone, cast it in bronze, print it on the memorial card. Everyone who scans it runs the same verified cell in their own browser and sees today's number. Free, global, for as long as the API and the care outlast the marker.

    Local: python3 -m http.server in the repo root, then open localhost:8000/showcase/demos/the-immortal-committer/ (crypto.subtle needs a secure context — localhost or https, not file://).
