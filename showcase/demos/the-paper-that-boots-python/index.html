    Why it's cool

    A printed square has no CPU, no OS, no battery, no network. Yet the instant a phone points at it, the phone becomes the computer: it reads a tiny base64 program out of the ink, proves the program wasn't tampered with (its SHA-256 must match the pin printed beside it, or it refuses), and then boots an entire language runtime — CPython 3.12, compiled to WebAssembly — inside a sandboxed iframe that can't touch your DOM, cookies, or storage. Two seconds later real Python is running code that lived only as ink. Nothing was installed. No app store approved it. No server is paying to keep it alive. Swap the payload and the same square ships any Python program — a calculator, a parser, a whole agent — to any device on Earth, forever.

    The verify step is not decoration. Press tamper above and watch: the moment a byte of the payload is rewritten, the recomputed hash no longer matches the pin, and CPython never boots. Your phone refused — no server told it to.

    The pattern · QR carries base64 Python + its pin

    The square is a /track-style trigger: instead of a URL it carries a <code>#<sha256> pair. The scanner page (this one) does the honest work:

    1. decode — base64 → the exact Python source bytes.
    2. verify — recompute SHA-256 in the browser (crypto.subtle, via the shared lib's sha256hex) and compare to the pin. Mismatch ⇒ refuse, before anything runs.
    3. boot — hand the sandboxed /vbrainstem-cell iframe the vbs-init handshake; it loads Pyodide and cold-boots CPython 3.12.
    4. eval — pass the verified source over a private MessagePort; CPython runs it and returns stdout.

    Run it yourself

    1. Open the raw runtime at /vbrainstem-cell — the same sandbox this page embeds — and type any Python into its eval box.
    2. Encode your own program with btoa("print('hi')"), pin it with its SHA-256, and print the pair as a QR with /track's zero-dependency encoder.
    3. Anyone who scans it boots the same verified CPython in their own browser. Free, global, forever.

    Local: python3 -m http.server in the repo root, then open localhost:8000/showcase/demos/the-paper-that-boots-python/ (Pyodide + crypto.subtle need a secure context and a network — localhost or https, not file://).
