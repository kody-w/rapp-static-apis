            Why it's cool

            This is a weld of a weld. The Sealed Storefront already gave you live availability, sealed order payloads, and custody hashes. Adding The Unbreakable Promise's ECDSA sign/verify loop upgrades completion into notarization: each order now emits a signed receipt tied to the exact sealed envelope and custody head. The buyer can verify signature authenticity without calling any server.

            The result is a no-backend marketplace with four trust layers in one browser session: live stock truth, confidentiality, tamper-evident transit, and non-forgeable seller attestation.

            Run it yourself

            1. Press Check what's available to pull live sold-out labels from GitHub Issues.
            2. Select at least one available item, add an optional note, then press Seal & submit order.
            3. Watch the order become an RSL1 envelope, then get SHA-256 stamped into the custody chain.
            4. Copy the emitted receipt JSON and verify it anywhere with crypto.subtle.verify using sellerPublicKeyJwk, receiptPayload, and signatureB64url.
            5. Use Verify full custody chain to replay the full provenance trail from genesis to head.

            Local: python3 -m http.server in repo root, then open localhost:8000/showcase/demos/the-notarized-marketplace/ (crypto.subtle needs HTTPS or localhost, not file://).
