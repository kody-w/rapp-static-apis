Proof of Freshness

Live — run it right now

Press run: your browser fetches a live payload, records the fetch timestamp, recomputes the SHA-256 of the exact response bytes with crypto.subtle, and extracts the number from those bytes. Then it runs runVerifiedCell() to verify the compute cell itself before execution and cross-check the same value path.

    Why it's cool

    A stale screenshot can fake a "live" number. This cannot. The proof includes two cryptographic anchors from your own browser: when the bytes were fetched and what exact bytes were fetched (SHA-256). Anyone can rerun it and compare. If the host ever serves different bytes, the hash changes instantly.

    And the compute step is verified too: runVerifiedCell() re-hashes the pinned module bytes before importing. So both the data proof and the code proof happen client-side, with no trusted server in the middle.

    Run it yourself

    1. Open DevTools Network and click 🧊 Prove freshness now.
    2. Copy the shown timestamp + SHA-256. Click again: timestamp always advances, and hash changes whenever payload bytes change.
    3. Inspect the narrated steps: cell: ✓ ... matches the pin confirms the compute cell was verify-before-exec checked too.

    Local: python3 -m http.server in the repo root, then open localhost:8000/showcase/demos/proof-of-freshness/ (crypto.subtle needs a secure context — localhost or https, not file://).
